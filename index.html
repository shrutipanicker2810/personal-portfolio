A Device ID bug in the client's app had been confidently wrong - reported, trusted, and acted on - until a cross-check between two data sources exposed it. Problem Framing Instinct

THE SYMPTOM IS NEVER THE QUESTION

MAD growth wasn't keeping pace with new activations. Something was eroding the base that the current metrics weren't capturing. Before touching any data, I had to decompose what "inactive" actually meant - was the device sitting unused in a room? Was inactivity seasonal, sector-specific, or a tenant walking away entirely? Each answer pointed to a different response. I worked with senior leadership to define the thresholds: dark at 28 days, churned at 90, reactivated if usage resumed after either. That framework gave customer success managers a precise enough picture to reach the right tenants and ask the right questions. I work backwards until I find a question worth answering. The right question doesn't just produce an insight - it produces an action.

In practice: Dark Devices Analysis · Decision Scientist, Mu Sigma, 2021
Applied AI Instinct

MEASURE AGAINST REALITY, NOT BENCHMARKS.

Every deployed voice assistant I used had the same problem - it was listening for silence, not for meaning. Fixed-threshold VAD treats every utterance identically, regardless of what the user is actually doing. I built a system to fix that: an intent-conditioned ensemble that classifies each utterance as transactional, informational, or reflective, then adapts its thresholds and scoring weights accordingly. If you're working through a decision out loud, the system gives you space. If the ensemble score crosses 0.85, it fires immediately - no point making the user wait for a threshold that no longer needs to be met. A model that's technically right but makes the user wait has still failed. AI isn't just about what the system gets right - it's about what the user actually experiences.

In practice: Semantic Turn Detection for Voice AI · AI Capstone CS6170, Northeastern University, 2026

Semantic Turn Detection for Voice AI
GitHub ↗

Most deployed voice AI systems use fixed-threshold silence detection - a design that ignores what the user is actually saying. I built an intent-conditioned turn-detection system that classifies each utterance as transactional, informational, or reflective, then dynamically adapts silence thresholds and scoring weights accordingly. The system combines a syntactic EOS scorer with an LLM-based semantic scorer in a weighted ensemble, with an early-fire override that responds immediately when the utterance is unambiguously complete. The system was designed as a three-condition ablation - VAD-only, syntactic EOS, and the full intent-conditioned ensemble, deployed as a real-time voice pipeline on Railway and evaluated in a within-subjects study with 18 participants across six counterbalanced orderings.

Naturalness 6.00/7 · Frustration 1.53/7 · Trust 5.70/7 - consistent improvement over VAD-only baseline

Photomosaic Generator
Live Demo ↗ GitHub ↗

A photomosaic generator that replaces per-image tile reloading with a session-level cache and per-pixel color matching with vectorized NumPy operations. The cache loads the 14,389-tile library once and reuses it throughout the session, eliminating redundant I/O. The vectorized matching computes color distances across the entire tile library simultaneously rather than iteratively. Both optimizations compound: less I/O overhead, faster matching, same output quality. Deployed on Hugging Face Spaces with an interactive Gradio interface supporting 16×16, 32×32, and 64×64 grid sizes with real-time comparison against the original implementation.

100–200× efficiency improvement over baseline via tile caching and NumPy vectorization · 0.553 SSIM at 32×32 optimal grid

Multi-Agent Combat Simulation with PPO and Genetic Evolution
GitHub ↗

Most multi-agent RL environments use fixed agent attributes - this simulation evolves them. We built a custom Pygame-based Royal Rumble environment from scratch after evaluating and rejecting MuJoCo's RoboSumo for its rigid reward structure and inflexible state space. The environment models 10 wrestlers with dynamic, static, and derived attributes competing in a turn-based combat system. Three complementary algorithms drive the simulation: Simulated Annealing selects initiators based on fitness while preserving match randomness, a Genetic Algorithm evolves weaker wrestlers through crossover and mutation between matches, and PPO trains each wrestler's combat policy across 10,000+ episodes. The result was emergent, wrestler-specific strategies rather than a uniform policy across all agents.

133% average population fitness improvement after one generation of genetic evolution · distinct emergent strategies per wrestler

Cancer Lymph Node Classification · ICIRCA 2020
Published · Jul 2020
2nd Intl. Conference on Inventive Research in Computing Applications

Most deep learning approaches to medical image classification treat preprocessing as a formality. This paper treats it as the core problem. We systematically evaluated 14 preprocessing and kernel initialization combinations on the PatchCamelyon dataset - 327,680 histopathological lymph node scans, to identify what actually helps a ResNet-50 detect metastatic cancer. Operator-based kernel initialization was identified and discarded early: it caused dangerous false negatives, classifying all images as non-cancerous, unacceptable in a medical context. The best configuration combined hematoxylin stain extraction with colour inversion using Glorot-initialized kernels, achieving 80.28% accuracy and 80.26% precision and recall. A heatmap analysis of misclassified images identified the failure mode: images with large white spaces or completely dark regions consistently confused the model.

80.28% accuracy · 80.26% precision and recall on 327,680 PatchCamelyon images · published ICIRCA 2020

Animal Segmentation in Non-Uniform Images · ICICCT 2020
Published · May 2020
4th Intl. Conference on Inventive Communication and Computational Technologies · LNNS Vol. 145

Nighttime CCTV footage of wildlife is a notoriously difficult segmentation problem - non-uniform illumination, excess noise, undefined boundaries, and occlusion up to 90%. We built a multi-stage pipeline: image enhancement via value thresholding, eye detection using FAST features to determine animal orientation, ROI extraction to isolate the subject, and systematic evaluation of three segmentation approaches - Region Adjacency Graphs, GrabCut foreground extraction, and DeepLab semantic segmentation. RAG sensitivity was analysed across segment count and threshold combinations. GrabCut demonstrated robust performance even under heavy occlusion conditions.
